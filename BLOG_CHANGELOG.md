## ✅ Что сделано

### Обложка `blog/index.html`
- Bento-мозаика вместо сетки из 20 одинаковых карточек: «Главное сейчас» (обзор цен), вертикальная плитка (Сочи), тёмная плитка «12 лет опыта», широкие плитки кейсов/гайдов, текстовые плитки инструкций — все 20 статей представлены.
- Плитка «Популярное» — топ-5 по реальным счётчикам просмотров.
- Фильтры по 12 рубрикам со счётчиками + живой поиск (`js/blog-filter.js`), состояние «ничего не найдено», `aria-pressed`.
- Глубокие ссылки `blog/#rub-<tag>` — обложка сама включает фильтр из hash.
- Хлебные крошки `Главная › Блог`.
- Убраны: эмодзи в мета, чёрные плашки «ИИ» на карточках (дисклеймер остался подписью под обложкой внутри статей).

### Все 20 статей
- Medium-стиль: светлый лист между тёмными шапкой/подвалом; тёмная шапка принудительно видна над светлым контентом.
- Липкий рейл слева: назад / Telegram / WhatsApp / копирование ссылки / кольцо прогресса чтения (`js/blog-article.js`).
- Визуальные хлебные крошки `Главная › Блог › Статья` (`aria-current`).
- Рубрика — ссылка-диплинк на фильтрованную обложку.
- «Коротко о статье»: якоря на ВСЕ разделы (H2 получили `id="sN"`), метрики чтения.
- Лид-подзаголовок, мета с `<time datetime>`, аватар редакции.
- Цветовая кодировка смысловых блоков (было: одинаковые highlight-box): совет = зелёный, важно = янтарный, пример = синий. 20 блоков перекрашены.
- Таблицы (2 статьи): сняты тёмные инлайн-стили, применён класс `post-table`.
- Конец статьи: «Вы дочитали», блок автора, «Дальше по теме» (2 карточки, приоритет той же рубрики), prev/next по датам, фирменный CTA.

### Инфраструктура

- `js/blog-filter.js`, `js/blog-article.js` — новые, `node --check` OK.
- `transform_blog_articles.py` — трансформер статей; **идемпотентен** (повторный прогон: 0 изменений, 20 пропущено).
- Документация: `BLOG_REDESIGN_PROPOSAL.md`, превью концептов в `blog-redesign-preview/`.

## 💡 Что можно добавить дальше (не входило в задачу)

1. **Визуальный QA в браузере** (Playwright/Chromium): скриншоты desktop + mobile, отлов мелких съехавших отступов — среда не смогла скачать браузер.
2. **Поиск по статьям на стороне индекса**: расширить `data-text` до полнотекстового индекса (сейчас — по title/рубрикам/кратким ключам) или подключить лёгкий движок типа Pagefind.
3. **Чтение вслух / «Слушать статью»** — кнопка в рейле (Web Speech API).
4. **FAQ-блоки с schema `FAQPage`** в подходящих статьях (шаблон вопрос-ответ уже стилизован в концепте C).
5. **Расширенные сниппеты**: добавить `Review`/`AggregateRating` данных компании в `Article` schema, `image` поле у BlogPosting (для карточек в выдаче).
6. **«Прогресс серии»**: группировка городских гайдов в серию с навигацией «часть 2 из 5».
7. **Автообновление плитки «Популярное»** из Яндекс.Метрики вместо статичных счётчиков (или обновлять числа просмотров скриптом).
8. **RSS/Atom фид блога** (`/blog/feed.xml`) и разметка для Яндекс.Дзена.
9. **Тёмная тема для светлого листа статьи** (переключатель день/ночь, `prefers-color-scheme`).
10. **A/B-замеры**: дочитываемость (scroll depth) до/после редизайна через Метрику — цель «читаемость» событиями.
11. **Генерация обложек статей** для новых плиток (часть плиток временно использует сервисные фото).
12. **Единый реестр рубрик**: сейчас 12 тегов — можно укрупнить до 6–7 (Советы, Города, Инструкции, Сравнения, Цены, Бизнес), был отдельный пункт в вопросах согласования.
