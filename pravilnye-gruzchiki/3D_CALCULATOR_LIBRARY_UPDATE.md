# Обновление Библиотеки Переезда - 3D Калькулятор

## Дата: 2026-07-18

## Изменения

### Проблема
Библиотека переезда показывала все предметы независимо от типа переезда (квартирный/офисный/коммерческий). Квартирные вещи (диван, кровать, стиральная машина) появлялись в офисном переезде, что было нелогично.

### Решение
1. **Добавлено поле `categories`** к каждому предмету в библиотеке
2. **Разделены предметы** по типам переезда:
   - `apartment` - квартирный переезд
   - `office` - офисный переезд
   - `commercial` - коммерческий переезд
3. **Добавлена фильтрация** в UI: библиотека показывает только предметы, подходящие для выбранного типа переезда
4. **Расширен список предметов** с 13 до 33 позиций

### Изменённые файлы

### Код изменений

#### 1. Добавлено поле `categories` к массиву `ge`
```javascript
ge=[
  {kind:"sofa",name:"Диван прямой",emoji:"🛋",...,categories:["apartment"]},
  {kind:"sofaOffice",name:"Диван зоны ожидания",emoji:"🛋",...,categories:["office"]},
  {kind:"box",name:"Коробка M",emoji:"📦",...,categories:["apartment","office","commercial"]},
  // ... остальные предметы
]
```

#### 2. Фильтрация в UI
```javascript
// Было:
ge.map(e=>c.jsxs("button",...))

// Стало:
ge.filter(e=>e.categories.includes(o||"apartment")).map(e=>c.jsxs("button",...))
```

#### 3. Обновлена функция приоритета сортировки
Добавлены новые виды предметов в функцию приоритета для корректной 3D-размещения:
- `sofaOffice`, `bedChild`, `tableOffice`, `chest` → приоритет 3 (крупная мебель)
- `chairsOffice`, `monitor`, `stroller`, `microwave`, `printer`, `coffeeMachine` → приоритет 4 (средние предметы)
- `rack`, `shopEquip` → приоритет 5 (паллеты и крупное оборудование)
