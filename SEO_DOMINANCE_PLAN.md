# Как сделать pragruz.ru любимчиком поисковиков

> Честный разбор: что надёжно, что рискованно, что нужно делать.

---

## Фундаментальная правда

Google не ранжирует страницы. Google ранжирует **ответы на запросы**.

Каждый запрос — это человек с проблемой. Поисковик даёт лучший ответ.
«Лучший» = самый полезный, самый полный, самый достоверный.

Сейчас pragruz.ru — это сайт с 86–93% шаблонным контентом между городами.
Это не «лучший ответ». Это «такой же ответ, как у 20 других сайтов, но с другим названием города».

---

## Два подхода: что надёжнее?

### Подход А: Широкий охват (много страниц, шаблонный контент)
- 5 городов × 16 услуг = 80 страниц
- Контент: шаблон + подстановка города
- FAQ: одни и те же вопросы, разное название города
- «Локальные условия»: 100 слов общего текста

**Риск:** Google Helpful Content / doorway-фильтр. Уже были обновления,
которые целенаправленно бьют по страницам, где «меняется только город».
Яндекс тоже борется с doorway-страницами.

**Надёжность: 🔴 Низкая.** Может работать сегодня, но каждый апдейт — риск.

### Подход Б: Глубокий охват (меньше страниц, уникальный контент)
- Каждая страница — **лучший ответ** на конкретный запрос
- Контент написан под конкретный город с реальным знанием местности
- FAQ отвечает на вопросы, которые реально гуглят
- Кейсы, цены, районы, нюансы — то, чего нет у конкурентов

**Риск:** Требует реальных усилий по созданию контента.

**Надёжность: 🟢 Высокая.** Это то, что Google прямо просит делать.
Каждое обновление алгоритма бьёт по шаблону и поощряет уникальность.

---

## ВЕРДИКТ: Подход Б — единственный надёжный

Но «меньше страниц» не значит «удалить страницы».
Это значит: **каждая существующая страница должна стать лучшей в своей нише**.

---

## ЧТО ДЕЛАТЬ: план по приоритетам

### Приоритет 1: Убрать doorway-риски (СРОЧНО)

Сейчас 86–93% пересечение контента между городами. Это красный флаг.

**Что делать:**

Для каждой городской страницы нужно добавить **минимум 400–600 слов УНИКАЛЬНОГО текста**, которого нет ни на одной другой странице сайта.

Что считать уникальным:
- Конкретные районы, улицы, ЖК с описанием особенностей
- Реальные кейсы: «Перееезд из ЖК X на ул. Y — 3 часа, 2 грузчика, Газель»
- Местные нюансы: пропуска, парковка, узкие дворы, сезонность
- Таблица цен по районам (ближе/дальше от центра — дороже/дешевле)
- Точное время подачи по районам
- Фотографии работ в ЭТОМ городе

Что НЕ считать уникальным:
- Подстановка названия города в шаблонную фразу
- FAQ, где отличается только «в Краснодаре» / «в Анапе»
- Один и тот же блок преимуществ на всех страницах

**Объём работы:** 5 городов × 16 услуг = 80 страниц × ~500 слов = ~40 000 слов уникального контента.

Это много. Но есть приоритизация — см. ниже.

### Приоритет 2: Контентная стратегия по страницам

Не все страницы одинаково важны. Вот приоритизация по поисковому спросу:

#### Уровень А — Высокий спрос (делаем ПЕРВЫМИ)

Эти страницы приносят 80% трафика. Каждая должна быть ЭНЦИКЛОПЕДИЕЙ своей темы.

| Страница | Запросы | Что добавить |
|---|---|---|
| **loaders** | грузчики + город | Цены по районам, калькулятор, кейсы 5+, фото бригад, видео процесса |
| **kvartirnyj-pereezd** | квартирный переезд + город | Пошаговый чек-лист, упаковка, типичные ошибки, кейсы по типам квартир (1к, 2к, 3к), время по районам |
| **ofisnyj-pereezd** | офисный переезд + город | Чек-лист для офис-менеджера, IT-оборудование, работа без простоя, B2B кейсы |
| **raznorabochie** | разнорабочие + город | Типы работ, требования к персоналу, оформление, смены, отличия от грузчиков |

**Глубина:** каждая страница уровня А — минимум 2000–3000 слов.

#### Уровень Б — Средний спрос

| Страница | Что добавить |
|---|---|
| gruzchiki-na-chas | Когда выгодно на час (vs на смену), конкретные задачи на 1-2 часа, цены |
| gruzchiki-na-smenu | Когда нужна полная смена, что входит, тарифы на смену vs почасовые |
| rigging | Типы такелажа (сейфы, банкоматы, станки, пианино), вес/цена, оборудование |
| vyvoz-musora | Виды мусора (строительный, бытовой, мебель), объёмы кузова, лицензии, полигоны |
| furniture | Виды мебели для сборки, время/цена, инструменты, гарантия |
| perevozka-mebeli | Упаковка, крепление в кузове, страхование, разобрать/собрать |

**Глубина:** 1500–2000 слов.

#### Уровень В — Низкий спрос / вспомогательные

| Страница | Что добавить |
|---|---|
| raznorabochie-na-sklad | Типы складских работ, требования, смены |
| raznorabochie-na-stroyku | Типы строительных работ, техника безопасности |
| arenda-gazeli-3m | Объём кузова, что влезает, цены, районы |
| arenda-gazeli-udlinennoj | То же + сравнение с 3м |
| autsorsing | B2B-фокус: договора, SLA, замена персонала |

**Глубина:** 1000–1500 слов.

### Приоритет 3: Городоспецифичный контент

Для КАЖДОГО города — уникальная секция, которая показывает реальное знание:

#### Краснодар
- Районы: ЦМР, КМР, ФМР, ЧМР, ЮМР, Гидрострой, Елизаветинская, Пашковка
- Пробки: Красная, Северная, Российская, кольцо Лукьяненко
- Новостройки: ЖК Парус, ЖК Панорама, ЖК Солнечный, ЖК Монарх — типичные заказы
- Проблемы: узкие дворы на Кропоткина, нет лифта в хрущёвках ФМР
- Сезонность: пик заказов март-октябрь

#### Анапа
- Районы: Центр, Джемете, Пионерский, Высокий, Супсех
- Сезонность: май-сентябрь — 80% заказов, курортный переезд
- Особенности: апартаменты без лифта, частный сектор с узким подъездом,
  мебель для доставки в квартиры посуточной аренды
- Проблемы: парковка у моря, узкие улицы в старой части

#### Сочи / Адлер / Сириус
- Районы: Центр, Хоста, Адлер, Сириус, Красная Поляна
- Сезонность: круглый год, пик — лето + Новый год
- Особенности: горный рельеф, серпантины, отсутствие подъезда к дому,
  многоэтажки без грузового лифта, апартаменты в Олимпийском парке
- Проблемы: пробки на Курортном проспекте, пропускной режим

#### Новороссийск
- Районы: Центральный, Южный, Мысхако, Цемдолина, Александровка
- Порт: разгрузка фур у терминалов — основная специфика
- Особенности: рельеф, ветра, промышленные зоны
- Проблемы: подъезд к портовым складам, пропускной режим

#### Геленджик
- Районы: Центр, Толстый мыс, Сосновка, Тонкий мыс
- Сезонность: май-октябрь, летний пик
- Особенности: перепад высот у моря, узкие подъезды, частные дома на склоне
- Проблемы: парковка на набережной, заезд в старый фонд

### Приоритет 4: FAQ — не шаблон, а реальные вопросы

Сейчас FAQ одинаковый для всех городов. Нужно для каждого города сделать
уникальные вопросы, которые люди реально гуглят.

**Примеры уникальных FAQ:**

Краснодар:
- «Сколько стоит подъём на этаж без лифта в хрущёвке ФМР?»
- «Работаете ли вы в пригороде — Елизаветинская, Пашковка?»
- «Можно заказать грузчиков на разгрузку фуры на базе Сормовская?»

Анапа:
- «Можно ли подъехать на Газели к дому в Джемете?»
- «Сколько стоит переезд из апартаментов в сезон?»
- «Работаете ли вы с посуточными квартирами — занос/вынос мебели?»

Сочи:
- «Нужны ли пропуска для Газели в Адлерский район?»
- «Можно ли перевезти пианино на 8 этаж без лифта в Сириусе?»
- «Работаете ли в Красной Поляне?»

Эти вопросы — это реальные длинные хвосты поисковых запросов.
Именно они приводят самых «горячих» клиентов.

### Приоритет 5: Блог — машина для длинных запросов

Сейчас 6 статей по 2500-4000 символов. Этого мало.

**Целевая структура блога:**

| Категория | Количество | Примеры тем |
|---|---|---|
| Чек-листы | 5-8 | «Чек-лист квартирного переезда: 47 пунктов», «Что купить для упаковки» |
| Кейсы | 5-10 | «Переезд 3-комнатной квартиры в ЖК Парус за 4 часа: разбор» |
| Сравнения | 3-5 | «Грузчики на час vs на смену: что выгоднее», «Газель 3м vs 4.2м» |
| Инструкции | 5-8 | «Как упаковать хрустальную люстру», «Как разобрать шкаф-купе» |
| Локальные | 5+ | «Особенности переезда в Сочи: пропуска, парковки, рельеф» |
| Цены/рынок | 3-5 | «Цены на грузчиков в Краснодаре: обзор 2026» |

**Каждая статья:** 5000-8000 символов, уникальная, с внутренними ссылками на услуги.

**Почему это работает:**
- Блог привлекает длинные запросы, которые не покрывают услуги
- Каждая статья — входная точка + перелинковка на услуги
- Google любит сайты, которые показывают экспертность через контент

### Приоритет 6: Доверие и E-E-A-T

Google оценивает Expertise, Experience, Authoritativeness, Trust.

**Что добавить:**

1. **Реальные отзывы с деталями** — не «Отличная работа, рекомендую», а
   «Переезд из 3-комнатной на Кропоткина 50 в новостройку на Сормовской.
   2 грузчика + Газель. Разобрали шкаф, упаковали, перевезли, собрали.
   4 часа, 4800 руб. Дата: 15 мая 2026»

2. **Фото с работ** — не стоковые, а реальные. Бригада грузит Газель,
   упаковка мебели, такелаж сейфа. 3-5 фото на страницу.

3. **О компании — детально** — лицензии, реквизиты, штат, годы работы,
   награды, партнёры. Всё, что подтверждает: это не сайт-однодневка.

4. **Цены открыто** — не «от 800 руб/час», а таблица:
   | Услуга | 1 грузчик | 2 грузчика | + Газель |
   |---|---|---|---|
   | 2 часа | 1600 | 3200 | 5200 |
   | 4 часа | 3200 | 6400 | 9400 |
   | Смена 8ч | 5600 | 11200 | 16200 |

   Конкретика = доверие. Клиент видит цену — звонит готовый.
   Это снижает отказы и повышает конверсию.

5. **Сертификаты/гарантии** — «Гарантия сохранности имущества»,
   «Компенсация при повреждении», «Договор с каждой бригадой»

### Приоритет 8: Schema.org — исправить адреса

Сейчас на ВСЕХ городских страницах указан краснодарский адрес:
```
"address": "ул. Кропоткина, д. 50, офис 339"
"addressLocality": "Краснодар"
```

Для Анапы, Сочи, Новороссийска, Геленджика — это вводит в заблуждение.

**Варианты:**
- А) Указать реальные адреса офисов/точек в каждом городе (если есть)
- Б) Убрать streetAddress, оставить только addressLocality
- В) Если офис только в Краснодаре — указать это честно:
  «Оперативный офис в [городе], юридический адрес: Краснодар, ул. Кропоткина, д. 50»

## КАК ДЕЛАТЬ КОНТЕНТ: практический подход

### Вариант 1: Полностью ручной (самый надёжный)
- Владелец/менеджер пишет тексты на основе реального опыта
- + контракт с копирайтером, который знает местную специфику
- Время: 2-4 часа на страницу
- Качество: максимальное

### Вариант 2: ИИ-черновик + экспертная вычитка (быстрее)
- ИИ генерирует структуру и базовый текст
- Эксперт добавляет: реальные кейсы, цены, адреса, нюансы
- Время: 30-60 минут на страницу (на вычитку и дополнение)
- Качество: высокое, если эксперт реально дополняет

### Вариант 3: ИИ без вычитки (НЕ рекомендую)
- Быстро, но шаблонно
- Google научился определять ИИ-контент без добавленной ценности
- Риск: Helpful Content penalty

**Рекомендация:** Вариант 2. Я генерирую черновик по структуре,
ты добавляешь реальные данные: кейсы, цены, адреса, нюансы, которые
знаешь только ты как владелец бизнеса.

## ИТОГОВАЯ ФОРМУЛА

```
Любимчик поисковиков = 
  уникальный контент (нет дубликатов)
  + экспертность (кейсы, цены, фото)
  + местная специфика (районы, улицы, нюансы)
  + технически чистый сайт (canonical, schema, speed)
  + регулярный блог (длинные запросы)
  + доверие (отзывы, реквизиты, гарантии)
```

Шаблон + подстановка города — это НЕ работает. Не сегодня, так завтра
поисковики это обнаружат. Единственный надёжный путь — каждая страница
должна быть **лучшим ответом** на запрос пользователя.
