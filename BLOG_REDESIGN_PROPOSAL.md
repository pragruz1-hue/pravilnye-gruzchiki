## Что сейчас не так (диагноз)

Раздел `/blog`: главная (`index.html`) + 20 статей. Весь блог выглядит однородной тёмной массой:

1. **Обложка — «стена» из 20 одинаковых карточек.** Нет избранной статьи, нет группировки, нет фильтров по 12 рубрикам. Порядок карточек не по дате (9 июля → 22 июня → 28 июня…).
2. **Статья — «простыня».** Светло-серый текст `#cbd5e1` на почти чёрном фоне `#07090e`, абзацы и списки визуально идентичны, нет содержания, резюме, таблиц, цитат.
3. **Все смысловые плашки одинаковые.** «Совет», «Важно», «Пример», блок услуг, CTA — все тёмно-синие с оранжевой каёмкой ⇒ читатель их игнорирует.
4. Хиро блога скопирован со страниц услуг (те же 2 CTA-кнопки) — не выполняет задачу обложки журнала.
5. Эмодзи 📅⏱👁 в мета-строках вместо системных иконок.
6. Чёрная плашка «Изображение сгенерировано ИИ» на каждой карточке — визуальный шум (дисклеймер нужен, но не так).
7. Нет содержания с якорями, прогресса чтения, резюме «коротко».
8. Конец статьи — тупик: нет автора и «Читайте также» (prev/next есть, но бледные).

## Три концепта (интерактивные превью)

| | A · «Экспертный журнал» | B · «Тёмный премиум 2.0» | C · «Bento + Medium» |
|---|---|---|---|
| Файл | `concept-a-journal.html` | `concept-b-dark.html` | `concept-c-bento.html` |
| Тема | Светлая «бумага» | Тёмная (как бренд) | Светло-серая + белые плитки |
| Обложка | Журнальная: избранная статья, нумерованная сетка, фильтры-чипы со счётчиками | Выбор редакции, липкая панель фильтров, секции «Свежее» / «Городские гайды» / «Инструкции» | Bento-мозаика, топ-5 «Популярное», плитки-цифры, поиск |
| Статья | Белый лист 720px, засечки, буквица, «Коротко за 30 сек», сайдбар-оглавление + CTA | Тёмная, прогресс-бар, оглавление слева, ghost-номера разделов, цветовой код блоков | Колонка 700px по центру, «В двух словах», липкий рейл с шарингом и прогрессом, FAQ-аккордеон |
| Риски | Визуальный разрыв с тёмным сайтом (решается общей шапкой/подвалом и тёмным CTA) | Тёмный фон для длинного чтения хуже светлого (компенсируется контрастом и ритмом) | Самое «немассовое» решение, сложнее в поддержке |

**Мой совет:** самый читабельный вариант — **A**; компромисс с брендом — **B**; можно смешать: **обложка B + статья A** (визуально совместимы).

## Общая система (вошла во все концепты)

- Карточки по дате убывания; избранная статья; фильтры рубрик со счётчиками (рекомендую укрупнить 12 тегов → 6–7 рубрик).
- Мета без эмодзи — SVG-иконки; AI-дисклеймер только как мелкая подпись под фото внутри статьи.
- В статье: содержание с якорями, резюме «коротко», цветовой код блоков (совет=зелёный, важно=янтарный, пример=синий, цитата=отдельный стиль), таблицы, шаги, автор, «Читайте также», prev/next, CTA.
- SEO/schema.org/URL/H1–H3 и внутренние ссылки сохраняются.

## План внедрения (после вашего «ок»)

1. Переписать `css/13-blog.css` под выбранный концепт (+ пару правок в базовых файлах при необходимости).
2. Переверстать `blog/index.html` (обложка: hero/featured/фильтры/секции).
3. Скриптом пройти по всем 20 статьям: добавить оглавление (генерируется из существующих H2), резюме, раскрасить блоки, мета-иконки, автор/related/prev-next. Тексты статей не меняем.
4. Проверка: sitemap, canonical, валидность, мобильная адаптивность, генерация недостающих правок.
